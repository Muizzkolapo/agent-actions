## v0.1.11 - 2026-04-13

### Enhancement or New Feature

* Cross-workflow chaining: --upstream and --downstream CLI flags, virtual action injection, and orchestrator support for multi-workflow pipelines
* NiFi-inspired FILE-mode record identity: tools receive full records with node_id, framework matches outputs to inputs by identity instead of heuristics (breaking change for FILE tools — use record["content"]["field"] instead of record["field"])
* Step-by-step progress display for sequential execution mode
* Add pre-commit hooks, rename wave tests, and fix docs site links
* Add --example flag to agac init for scaffolding projects from bundled examples
* Add doc-vs-code audit framework for retry and reprompt modules, remove undocumented features from docs

### Bug Fix

* Fix FILE-mode lineage: preserve content from full records returned by FILE tools, and skip source data loading in cross-workflow runs
* Resolve .env files from project root instead of CWD, fixing env var loading in worktrees and subdirectories
* Guard conditions can now reference output_field values directly (e.g., `severity != "low"` instead of `assess_severity.severity != "low"`), with improved error messages suggesting dot-notation paths for unresolved fields

## v0.1.0 - 2026-03-27

Initial release.

### Features

- Declarative YAML workflow definition with per-action model, context, and schema configuration
- Multi-provider LLM support: OpenAI, Anthropic, Google Gemini, Groq, Mistral, Cohere, Ollama
- DAG-based dependency resolution with automatic parallel execution
- Batch processing via provider batch APIs for high-throughput workloads
- Pre-check guards to skip actions before making LLM calls
- JSON schema validation with automatic reprompt on malformed output
- Context scoping (`observe` / `drop`) to control token usage per action
- User-defined functions (UDFs) for custom pre/post-processing logic
- Parallel consensus pattern (multiple voters + merge)
- CLI (`agac`) with `run`, `batch`, `init`, `inspect`, `docs`, and `schema` commands
- LSP server (`agac-lsp`) for editor integration with diagnostics and completions
- Interactive documentation site via `agac docs`
- Per-action observability: timing, token counts, and structured event logs
